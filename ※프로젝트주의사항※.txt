<깃 허브 사용방법>
깃 허브란 ? 구글 드라이브, 네이버 클라우드 같은 것이라고 생각하면 됨.
	프로젝트를 온라인 상에 저장할 수 있는 원격 저장소.

Pull : 원격 저장소에 있는 프로젝트를 다운 받는 것 ( N드라이브에서 사진 저장하는거라고 생각하면 됨)
Push : 로컬 저장소에 (= 내 컴퓨터) 저장되어 있는 프로젝트를 수정한 뒤 원격 저장소에 업로드 하는 것.
Fetch : 새로 고침 (Push, Pull 전에 꼭 수동으로 패치해서 Pull 내역이 있는지 확인 바람)
Commit : 로컬 저장소에서 생긴 변경 사항을 Push 전 저장해 놓는 것.

순서 : Fetch -> Pull -> 로그 작성 -> Commit -> Push

"Pull 방법:
1. 새로 고침 : Fetch를 통해 Pull 받을 내역이 있는지 확인
2. 풀 받기 : 받을 내역이 있다면 Pull 받기.

"Push 방법"
1.로그 작성 : Summary 에 작업 내역, 변경 내역 등등 작성 (부연 설명이 필요하다면 Description 작성)
2.커밋 하기 :Commit to main 
3.푸시 하기 : 오른쪽 상단의 Push 누르면 됨.


<깃허브 사용 시 주의사항>
- 깃 허브를 사용할 경우, 현재 내 컴퓨터에 저장 되어 있는 프로젝트의 상태와 원격 저장소의 상태가 다르면 ★충돌★이
발생할 수 있음.

- 어느 부분을 작업할 것인지, 변경 전 꼭꼭꼭꼭꼭 카톡으로 얘기해줘야 함.
- 작업 시작 전,후로 Fetch-> Pull 필수.
=> 안그러면 상대방이 작업했던거 전부 날라갈 수도 있음


문제가 생기거나 잘 모르는게 있으면 숨기지 말고 언제든 민아에게 연락 주삼
자세한 사용법은 나중에 디코로 알려드림


<폴더 관련 주의사항>
- 파일명은 꼭 약속한대로 표기 부탁 바람

- 경로, 파일명에 한글 절대 금지 절대 금지 금지 금지 한글쓰면 죽음뿐이야

-폴더, 파일 읽어들이기 등의 행위는 경로와 파일명을 기반으로 동작하므로, 함부로 폴더 위치를 변경하거나,
 파일명 절대 절대 절대 건들면 안됨 ! (필요하다면 민아에게 꼭 얘기 부탁 바람)

- 프로젝트 Pull ,Push Commit 등에 문제가 생겼다고 해서 절대로 어떠한 폴더도 함부로 삭제하고 푸시하면 안됨!
